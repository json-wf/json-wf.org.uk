JSON-WF - a blogroll interchange format

JSON-WF (JSON for Web Feeds) is a blogroll interchange format that is to JSON what OPML is to XML.

Why?

Most online data interchange these days (with APIs being a good example) is done using JSON, possibly with a schema to describe expected keys, values and types. There are far more JSON parsing libraries for every possible programming or scripting language than there are XML parsing libraries, and they tend to be smaller and faster too due to the more limited feature-set.

Why not OPML?

OPML has been around for a long time, and is a de-facto standard supported by most feed readers for importing and exporting blogrolls. However, its design is much more complex than required for this purpose, and indeed most of that complexity is used not for feed readers but for outliners, its primary focus. The goal is to create a dedicated format for blogroll sharing.

Where do I start?

The JSON-WF specification is very new, so there are currently no known implementations in feed readers. The advice is - please wait! The aim is to encourage developers of feed readers and other associated applications to add support in time.

If you run one or more sites that currently use OPML to share blogrolls with readers, please consider also publishing an equivalent JSON-WF file.

Read the JSON-WF specification to start building or adding support for JSON-WF blogrolls to your website or application.
